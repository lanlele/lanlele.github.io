1002 商业需求文档BRD

发表于 2017-09-08 | 分类于 1000 产品策划

1.文档的目标用户

领导、老板、投资者等。关注点：1、需要投多少钱／资源；2、能挣多少钱；3、风险能否承担。重点突出商业价值和资源评估，即性价比。

2.项目背景

2.1明确用户场景和问题

用户在什么样的场景下遇到什么样的问题。

2.2定位自己，找到我们在哪儿

明确是否到了合适的时机切入市场、较竞争对手是否有足够的优势、自身是否有足够的资源来实施。最好有相应的数据进行支撑。

2.2.1整个行业如何——市场扫描

PEST分析，分别分析政治法律环境（Political Factors）、经济人口环境（Economic Factors）、社会文化环境（Social Factors）、技术环境（Technological Factors）四个方面的机会和威胁。

2.2.2竞争对手如何——竞品分析

上网搜、看行业分析报告、做调研等，收集尽可能多的、对的信息，进而作出尽可能正确的决策。

2.2.3自己情况如何——自我分析

SWOT分析，包括分析企业的优势（Strength）、劣势（Weakness）、机会（Opportunity）和威胁（Threats）。因此，SWOT分析实际上是将对企业内外部条件各方面内容进行综合和概括，进而分析组织的优劣势、面临的机会和威胁的一种方法。

2.2.4竞争优势

  1. 现有优势是指我们做这个产品之前就具备的优势，比如可以包括：市场先行者、专利、成本优势、价格优势、牢靠的合作伙伴、市场联盟、良好的客户关系，以及专业的商业管理团队。
  2. 获得优势是指我们做这个产品之后能够具备的优势，比如可以包括：市场先行者、市场占有率、击败竞争者和建立领导地位等。
  3. 产品固有优势：产品本身的固有优势。
  4. 竞争壁垒：说明我们如何建立自己的竞争壁垒和竞争对手给我们建立了什么竞争壁垒。

3.商业价值

BOSS最感兴趣的点，做了这个项目以后有什么价值，可以预测一下相关数字的变化。需要明确我们去哪里？需要具体明确的有：细分市场是什么？目标用户是谁？我们要解决他们的什么问题，满足他们的什么需求？ 安索夫矩阵、BCG矩阵、波特五力模型、GE矩阵、战略地图、SPAN战略定位分析（Strategy Positioning Analysis）、价值链分析法等

4.功能需求描述

将筛选出来的需求通过功能点进行表达，并画出业务逻辑关系。解决我们怎么去？需要进一步明确的问题有：用什么产品满足需求？产品的核心竞争力是什么？

4.1市场战略

  1. 比竞争者提供更多的价值。
  2. 有助于建立可持续的竞争优势。
  3. 传播良好的感知价值。

4.2营销策略

  1. 目标客户：定义谁会购买产品，以及他们购买产品的主要原因。
  2. 产品定位：定义并描述产品定位。
  3. 产品：定义并描述产品的物理特征、质量、功能，以及带给客户的价值。
  4. 定价：定义并描述你如何向客户收费。描述价格模型和影响因素。明确竞争者的价格。
  5. 促销：定义并描述用来形成购买意识、了解产品差异化和产品需求满足能力的促销、奖励、广告、活动、公共关系，以及所有营销的过程。
  6. 渠道：定义并描述用来把产品提供给客户的销售和分销渠道。

4.3辅助战略

  1. 成长战略：定义并描述任何适用产品成长的途径——产品多样化，产品持续开发，市场持续开发，以及市场多样化。
  2. 融合战略：定义并描述任何适用产品线融合的途径——市场收缩、产品或产品线特征减少，以及产品终止。

5.非功能需求描述

数据、性能、产品服务、营销、安全、接口、法务、财务、帮助等方面加以描述。

6.资源评估

第二个重点！BOSS们要看成本，他们在了解达成项目的目标需要多大的花费以后，才能做出决策。

7.风险和对策

风险因素可能有：决策改变；优先级冲突；缺乏专业能力；缺乏预备预算；缺乏必需技术；缺乏高级管理层的支持；缺乏自由支配关键商业资源的能力；缺乏足够的成本和时间向业务团队提供支持。
